# 🌱 EcoTrack & IA — Formation Intensive 2026

> **Transformer le cycle de vie logiciel (SDLC) avec l'Intelligence Artificielle Générative dans l'écosystème Microsoft**.

## 📖 Le Pitch

Bienvenue dans le parcours de formation **EcoTrack & IA**. Ce cursus intensif a été conçu pour unifier les équipes techniques et métiers autour d'un cas d'usage réaliste, en exploitant le plein potentiel de l'**Intelligence Artificielle Générative (GenAI)** dans l'écosystème Microsoft.

L'objectif n'est pas seulement d'apprendre à coder plus vite, mais de **transformer l'ensemble du cycle de vie logiciel (SDLC)** : de l'expression du besoin métier au déploiement en production, en passant par l'analyse de données et la sécurité.

---

## 🏭 Le Projet Fil Rouge : "EcoTrack"

**EcoTrack** est une startup fictive développant une plateforme **SaaS B2B innovante**.

Face aux nouvelles réglementations environnementales (**CSRD**) et aux enjeux du **Numérique Responsable (Green IT)**, EcoTrack permet aux **DSI** et aux directions **RSE** de :
- 📊 Mesurer l'empreinte carbone de leur parc informatique
- 🔍 Analyser les données d'émissions (serveurs, laptops, cloud)
- 🎯 Réduire activement leur impact environnemental

Tout au long des **10 modules**, les participants construisent le **MVP (Minimum Viable Product)** de cette application, chacun selon son domaine d'expertise, en se servant de l'**IA comme d'un "Pair" de très haut niveau**.

## 🗺️ Parcours d'Apprentissage — Les 10 Modules

### 🟢 **Module 1** : Conception & Architecture (Tech)
**Focus** : "Shift Left" — Modélisation relationnelle SQL Server, génération de scripts DDL optimisés via l'IA.

### 🤝 **Module 2** : Spécifications & Analyse Métier (Biz)
**Focus** : Combler le fossé Métier/Dev — Traduction de User Stories en BDD (Gherkin), création d'Agents M365 personnalisés.

### 🟡 **Module 3** : Développement & Agents Simples (Tech)
**Focus** : Implémentation du moteur C# avec GitHub Copilot, refactoring de SQL legacy.

### 🤖 **Module 4** : Création d'Agents Spécialisés (Tech)
**Focus** : Définition de 4 profils d'IA stricts via des fichiers `.agent.md` (Unit Tester, Refactoring, API Contract, Security Review).

### 🧠 **Module 5** : Orchestration Multi-Agents (Tech)
**Focus** : L'automatisation du QA — Master Agent orchestrant les sous-agents pour auditer et corriger le code.

### 🎨 **Module 6** : Frontend & Visualisation (Tech)
**Focus** : Génération de composants UI (Dashboards), standardisation CSS via des fichiers "Skills".

### 🛡️ **Module 7** : Sécurité Ciblée - AppSec (Tech)
**Focus** : TDD avec xUnit/Moq, audit de failles de sécurité classiques.

### ⚙️ **Module 8** : DevOps & Déploiement (Tech)
**Focus** : Infrastructure as Code assistée — Pipelines GitHub Actions, Docker Compose avec PostgreSQL.

### 📊 **Module 9** : BI, Data & Reporting (Biz)
**Focus** : L'analytique avec l'IA — Datasets paramétrés SSRS, calculs DAX, Storytelling pour le management.

### ⚫ **Module 10** : Documentation (Tech)
**Focus** : Génération du README de référence à partir du projet.

## 🎯 Philosophie & Acquis de la Formation

À l'issue de ce parcours, les participants auront compris que l'IA ne se limite pas à un "Chat" dans le navigateur.

### 🔑 Concepts Clés Assimilés

1. **Context Grounding**  
   Fournir le bon contexte (MCD, fichiers ouverts, lexique métier) est la clé d'une réponse IA pertinente.

2. **Orchestration d'Agents (Multi-Agent Systems)**  
   Découper les problèmes complexes et les confier à des IA spécialisées pilotées par un orchestrateur.

3. **Infrastructure via "Skills"**  
   Figer les conventions d'entreprise (Règles de sécurité, stacks) dans des fichiers de référence ingérés par l'IA.

4. **Collaboration Transversale**  
   L'IA utilise les livrables d'une équipe comme contexte d'entrée direct pour générer le code de l'autre équipe.
